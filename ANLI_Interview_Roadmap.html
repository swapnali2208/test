Dockerfile

FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY src/ ./src/
COPY configs/ ./configs/
COPY notebooks/ ./notebooks/

CMD ["python", "src/predict.py"]

docker-compose.yml

version: "3.8"
services:
  anli-classifier:
    build: .
    volumes:
      - ./data:/app/data
      - ./results:/app/results
    environment:
      - MODEL_PATH=/app/models/best_checkpoint

Key Tips for Standing Out

★ Show Your Reasoning
  • Explain why you chose each model, not just what you did
⚠ Acknowledge Adversarial Nature
  • ANLI examples were crafted to fool models — 50–60% on R2 is respectable
✓ Clean Code
  • Modular functions, type hints, docstrings
↺ Reproducibility
  • Set random seeds everywhere, document full environment
✎ Notebook Quality
  • Markdown headers, clear sections, inline plots
🔎 Error Analysis
  • Go deep on why the model fails — this separates strong candidates
